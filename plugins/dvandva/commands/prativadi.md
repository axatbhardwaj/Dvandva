Serialize all outputs via the single-writer rule — only the parent role writes the baton, and generated agents never own `assignee`, `active_roles`, phase transitions, or final approval. Generated instances are run-scoped and ephemeral; no additive roster sprawl unless promoted. `seed_agent` is advisory provenance, not the executable gate.
