## Dynamic agents (seed roster)

When a phase needs more parallel capacity, plan dynamic tracks in `work_split`, generate briefs from the seed roster (each brief satisfies the same agent contract as its seed agent), record instances in `agent_instances` on the baton, dispatch harness subagents, and apply explicit closure: close handles and record closure evidence plus non-empty `work_item_ids` before any track counts as completed. Serialize all outputs via the single-writer rule — only the parent role writes the baton, and generated agents never own `assignee`, `active_roles`, phase transitions, or final approval. Generated instances are run-scoped and ephemeral; no additive roster sprawl unless promoted. `seed_agent` is advisory provenance, not the executable gate. Coordination invariant: no daemon, no hidden orchestrator — the baton is the only coordinator. Path invariant: dynamic write-path disjointness — non-empty `write_paths` must be pairwise disjoint for generated instances sharing the same `base_checkpoint`, or for any two live (`planned`/`running`) instances regardless of base_checkpoint, unless serialized through `depends_on` within a shared `conflict_group`; closed instances from an earlier base_checkpoint do not block later sequential reuse.
